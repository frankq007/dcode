# DCode App 测试指南

## 测试环境准备

### 1. 启动后端服务

在运行 App 测试之前，需要确保以下服务正在运行：

#### 启动 Mock OpenCode 服务器
`ash
cd D:\code\dcode\mock-opencode
npm run dev
`
服务将在 http://localhost:3000 运行

#### 启动 Gateway（直连模式）
`ash
cd D:\code\dcode\gateway
npm run dev
`
Gateway 将在 http://localhost:8765 运行

#### 启动 Relay Server（可选，云转发模式）
`ash
cd D:\code\dcode\relay
npm run dev
`
Relay 将在 http://localhost:8766 运行

### 2. 启动 HarmonyOS 模拟器

使用 DevEco Studio 启动 HarmonyOS NEXT 模拟器（API 12+）

### 3. 构建并运行 App

在 DevEco Studio 中打开 App 项目，构建并部署到模拟器

### 2. ChatPage（聊天页）

## 功能集成测试

### 集成测试 1：完整的直连流程

1. **启动 Gateway（直连模式）**
   `ash
   cd gateway
   npm run dev
   `

2. **在 App 中扫码连接**
   - 打开 App
   - 点击"扫码连接"
   - 扫描 Gateway 终端的二维码
   - 点击"连接并保存"

3. **发送消息并接收响应**
   - 在 ChatPage 输入 "你好"
   - 点击发送
   - 等待 AI 回复（包含 Markdown 和 Mermaid）
   - 验证所有消息类型正确渲染

4. **处理权限请求**
   - 等待 Gateway 发送权限请求
   - 点击"允许"
   - 验证请求被正确处理

5. **验证连接保存**
   - 返回 ConnectionListPage
   - 验证连接已保存并显示在线状态

### 集成测试 2：完整的云转发流程

1. **启动 Relay Server**
   `ash
   cd relay
   npm run dev
   `

2. **启动 Gateway（Relay 模式）**
   - 修改 gateway.config.json：mode = "relay"
   `ash
   cd gateway
   npm run dev
   `

3. **在 App 中扫码连接**
   - 打开 App
   - 点击"扫码连接"
   - 扫描二维码（包含 Relay URL）
   - 点击"连接并保存"

4. **验证通过 Relay 连接**
   - 检查 Relay 控制台日志
   - 验证消息通过 Relay 转发

5. **测试断开重连**
   - 停止 Gateway
   - 观察 App 显示"未连接"
   - 重启 Gateway
   - 验证 App 自动重连

### 集成测试 3：会话管理

1. **创建多个会话**
   - 在 ChatPage 点击菜单 → "新建会话"
   - 重复 3 次创建 3 个会话

2. **在不同会话间切换**
   - 点击顶部会话名称
   - 选择不同会话
   - 验证每个会话有独立的聊天历史

3. **验证会话列表同步**
   - 检查 Gateway 控制台
   - 验证 session_list 消息正确发送

## 已知问题和限制

### 当前限制
1. **语音输入**：使用 Mock 实现，实际 ASR API 需要真机测试
2. **Mermaid 渲染**：依赖外部 CDN，需要网络连接
3. **Markdown 渲染**：自定义实现，部分高级特性可能不支持

### 待优化项
1. 消息列表虚拟滚动（大量消息时）
2. Mermaid 图表缓存机制
3. 离线消息队列
4. 推送通知支持
